### **追加要件1：コマンド直前の環境変数代入 (Variable Assignments)**

Bashでは、コマンドの前に VAR=value を置くことで、**そのコマンドの実行時のみ有効な環境変数**を設定できます。

**例:** LANG=C ls -l

**パース要件:**

Parserは、コマンド名（ls）が来る**前**に、= を含む単語（WORD）が来た場合、それを「引数」ではなく「環境変数定義」として区別して保持する必要があります。

通常の引数 (args) とは別のリスト (assigns など) に格納します。

**実装への影響:**t_node 構造体にメンバを追加する必要があります。

typedef struct s_node {
// ...
char **args;      // ["ls", "-l"]
char **assigns;   // ["LANG=C", "DEBUG=1"]  <-- これ！
} t_node;

## **追加要件2：ヒアドキュメントの「変数展開フラグ」の判定**

ヒアドキュメントには隠れたルールがあります。**終了文字（デリミタ）をクォートしたかどうかで、中身を変数展開するかどうかが決まります。**

**ケースA:** cat << EOF

中身の $USER などは**展開されます**。

**ケースB:** cat << "EOF" （または 'EOF'）

中身の $USER は**展開されず、そのまま文字として扱われます**。

**パース要件:**

Parserは << の次のトークン（デリミタ）にクォートが含まれているかチェックします。

クォートが含まれていれば、デリミタからクォートを削除（"EOF" → EOF）しつつ、**「中身を展開しない」というフラグ**をノード（またはリダイレクト情報）に立てます。

クォートがなければ、**「中身を展開する」というフラグ**を立てます。

**実装への影響:**
パース時にヒアドキュメントの内容を一時ファイルに書き込む際、このフラグを見て「書き込む前に $ を展開するかどうか」を決定します（あるいは実行時に展開処理をするなら、フラグだけ保持しておきます）。
